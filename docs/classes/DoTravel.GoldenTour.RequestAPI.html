Parameters

array $customerINFO

START CONTENTS INSIDE CUSTOMER title The title of Customer like Mr., Miss., Dr. etc. It is required field. firstName The first name of customer. It is required field. lastName The last name of customer. It is required field. email The Email address of customer. It is required field. address1 The address 1 of customer. It is required field. address2 The address 2 of customer. city The city of customer. It is required field. county The county of customer. postCode The post Code where customer lives. It is required field. countryCode The code of country where customer lives. mobile The mobile number of customer.

If customer wants to receive newsletter send 'Y' for yes else 'N' for no. If customer wants an Email of Booking then send 'Y' or 'Yes' else 'N' or 'No'.
array $productINFO The container where all the details of product will be passed. If ticketRefNumber will be inserted then this ticket number will be used otherwise it will be generated automatically. Format of travel date should be dd/MM/yyyy.(When product type is add on and flag allowed date is 'N' then it is not required. Also in attraction Open Dated ticket it is not required.) Unique Id assigned to product. Please see (productdetails.aspx). Unique Id of schedule. Please see (productdetails.aspx). The unique Id of Pick up and Time which is selected. Please see (productdetails.aspx). The unique Id of Drop Point which is selected. It required to pass for Shuttle Transfer Type Product. The pax token which will received by paxreservation API product wise. Pass reference number.It is not required field. The special requirement of customer for the product selected. In case of only 'Hotel' product type it is required field. The promotional code of the customer. The main container of all the pax(s). The container which will contain the unitId and paxCount. unitId is the unique Id of each price unit (Adult, Child, etc.). Please see (productdetails.aspx). It is required field. The quantity of each unit. It is required field (Only integer value is accepted). When product type is transfer then this container is required and all the tags inside the tag are required. Name of airline. Number of flight. Time of transfer. Format of time should be HH:MM AM/PM (For Shuttle type transfer product) Name of hotel. Address of hotel. Postcode of hotel. Mobile number of Customer. Origin. Destination. The name of person which is to be transferred.
String $currencyCode

Pass any currency code from following table.
String $paymentMode

'C' or 'A' (C = Credit card and A = On Account).
array $cardPaymentINFO Name which is on credit card. Credit Card Number. Enter any type from following table. Expiry date of credit card in mm/yyyy format. Issue number of credit card. Card Verification number of credit card. 'Y' or 'N' (Y = 3DEnroll for transaction and N = No 3DEnroll for transaction ). Website address where should be redirection performed from bank's website after 3DEnroll. This address must be XML agent's website URL.
